# 🎉 AstrBot E2B 云沙箱插件更新日志

### 🌟 v1.2.0 重大架构升级：持久化沙箱与权限控制
* **💾 会话级持久沙箱**：告别“即用即焚”！同一个聊天会话现在能“记住”你的沙箱状态，变量和文件都在。大模型现在可以像操作真实电脑一样，自主创建、暂停、恢复、销毁沙箱。
* **🛡️ 白名单访问控制**：新增 `user_whitelist` 功能，只让指定 ID 的用户使用插件，你的沙箱算力你做主！
* **🎯 文件发送权交还模型**：生成的文件不再强制刷屏。结果会先静默进入本地缓存队列，由模型调用专用工具精准挑选并下发“最正确”的文件，彻底告别错发漏发。
* **💰 自动暂停与成本管控**：默认执行完毕即自动暂停 (`auto_pause=true`)。仅在模型明确需要执行长效任务时才保持后台运行，精细化避免云端额度浪费。

### 🕰️ 往期历史更新回顾

**v1.1.0 | 文件回传修复与稳定性增强**
* **📊 Excel 损坏彻底修复**：将回传链路改成按原始字节下载，沙箱生成的 `.xlsx` 文件打开再也不会提示“已损坏”，**Excel 终于不炸了**！
* **🚫 坏文件拦截**：新增 ZIP 容器类文件完整性校验，发现损坏直接跳过，拒绝把坏文件硬发给用户。
* **🧹 缓存自动清理**：给会话级文件缓存加了过期清理机制，避免插件跑久了把你的硬盘塞满。

**v1.0.9 | Template 支持与回传增强**
* **Template 接入**：新增 `default_template`，支持默认从你自己准备好的 E2B 模板启动沙箱。
* **精准筛选**：优先筛选本次执行里真正新建或修改过的结果文件。

**v1.0.8 | 文件回传与稳定性优化**
* **本地缓存池**：沙箱生成文件先存入插件本地目录，完美绕过“无法直接发送绝对路径”的报错。
* **防呆提示**：给大模型增加了系统级警告，防止模型在多轮对话中乱调不存在的变量。

**v1.0.7 | 文件桥接初版**
* **会话附件接入**：支持识别用户刚刚上传的文件，自动传到沙箱供代码读取。
* **渠道兼容**：除 NapCat 外，现已完美兼容 WebChat 渠道的文件解析。

**v1.0.6 | 配置修复与兼容增强**
* 修复了配置文件格式，让 `max_output_length` 真正生效，防止超长输出刷屏。
* **代理支持**：新增 `proxy` 配置，拯救国内直连 E2B 的网络困难户。

**v1.0.5 | 性能与超时调整**
* 移除了部分不常用库的自动安装逻辑，沙箱启动更快。
* 延长默认执行超时时间，给复杂计算留出余地。

**v1.0.4 | 核心修复与体验优化**
* **杜绝僵尸沙箱**：引入服务端自动销毁机制，彻底解决沙箱残留导致的无意义扣费问题。
* **告别缺库报错**：智能检测代码中的常用库并自动 `pip install`。
* **中文画图支持**：自动注入中文字体，解决画图变成“豆腐块”乱码的问题。

**v1.0.3 | 逻辑闭环与稳定性提升**
* **打通 LLM 闭环**：代码运行结果回传给大模型，Bot 终于知道自己算出了什么。
* **防死循环机制**：引入代码哈希检测与系统指令强注入，彻底解决大模型反复调用工具导致的 Token 浪费。

**v1.0.0 - v1.0.2 | 梦开始的地方**
* **v1.0.2**：买了一只笔，学会了画图，底层新增图片自动捕获机制。
* **v1.0.1**：修复了沙箱中没有 AstrBot 的 bug (加了个图片 Logo)。
* **v1.0.0**：在与 AI 评审员的深度交流中茅塞顿开，推出了第一个能跑的版本。
