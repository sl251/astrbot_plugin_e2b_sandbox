# 更新日志 (Changelog)

- **v1.1.0** **文件回传修复与稳定性增强**
  - **Excel 损坏问题修复**：之前沙箱里生成的 `.xlsx` 文件虽然能发出来，但在本地打开可能会提示“文件已损坏，无法打开”。这次把回传链路改成了按原始字节下载，Excel 终于不再炸了。
  - **坏文件拦截**：新增 ZIP 容器类文件完整性校验，发现损坏就直接跳过，不再把坏文件硬发给用户。
  - **自动回传恢复**：为了兼容部分 `new api / one api` 对函数调用 schema 的限制，当前版本恢复为“插件自动选择并发送最佳结果文件”的模式，减少额外工具带来的兼容性问题。
  - **缓存清理补上**：给会话级文件缓存和发送记录加了过期清理，避免插件跑久了缓存越堆越多。

- **v1.0.9** **Template 支持与结果回传增强**
  - **Template 配置接入**：新增 `default_template` 配置项，支持默认从你自己准备好的 E2B 模板启动沙箱。
  - **结果文件筛选增强**：生成文件会先下载到本地 `exports/`，并优先筛选本次执行里真正新建或修改过的结果文件。
  - **文档补充**：补充了 E2B Template 的使用说明、成本说明和一个最小模板示例。

- **v1.0.8** **文件回传与稳定性优化**
  - **本地回传缓存池**：沙箱内生成的文件现在会先下载到插件本地的 `exports/` 目录中，完美绕过部分环境下“无法直接发送沙箱绝对路径”的报错。
  - **智能筛选回传文件**：优先筛选“本次执行里新建或修改过”的文件，并自动过滤空文件、重复文件，命中率大幅提升。
  - **“即用即焚”防呆提示**：给 LLM 增加了系统级警告，明确告知沙箱是无状态、单次调用即销毁的，防止模型在多轮对话中“无中生有”调用不存在的变量。
  - **静默下载**：优化了内部字体下载机制，减少 `stderr` 中无意义的进度条污染。

- **v1.0.7** **文件桥接初版**
  - **会话附件接入**：支持从 AstrBot 会话中识别用户刚刚上传的文件，并自动上传到沙箱供代码读取（默认传至 `/home/user/uploads`）。
  - **兼容性增强**：除了 NapCat，现已兼容 WebChat 渠道下的文件组件解析。

- **v1.0.6** **配置修复与兼容增强**
  - 修复了损坏的 `metadata.yaml` 与 `_conf_schema.json`。
  - 修复了 `max_output_length` 成为摆设的问题，现已真正参与结果截断，防止输出过长刷屏。
  - **代理支持**：新增 `proxy` 配置，拯救国内直连 E2B 的网络困难户。
  - **结果采集增强**：更完整地收集 `stdout`、`stderr`、文本结果与执行崩溃的错误栈。

- **v1.0.5** **性能与超时调整**
  - 移除了一些不常用库的自动安装逻辑，提升沙箱启动速度。
  - 延长了默认的执行超时时间，给复杂计算留出余地。

- **v1.0.4** **核心修复与体验优化**
  - **杜绝僵尸沙箱**：引入服务端自动销毁机制（`timeout` 参数），彻底解决沙箱残留导致的无意义扣费问题。
  - **依赖自动安装**：智能检测代码中的常用库并执行 `pip install`，告别 `ModuleNotFoundError`。
  - **中文画图支持**：自动注入中文字体，解决 Matplotlib 绘图变成“豆腐块”乱码的问题。
  - **SDK 适配**：修复了新版 E2B SDK 参数更改导致的报错问题。*(感谢 @xboHodx 和 @IMAUZSA 的反馈建议以及 @LiYH2008 的鼓励！)*

- **v1.0.3** **逻辑闭环与稳定性提升**
  - **打通 LLM 闭环**：代码运行结果现在会回传给大模型，Bot 终于知道自己算出了什么，并能对结果进行解释。
  - **防死循环机制**：引入代码哈希检测与系统指令强注入，彻底解决了大模型反复调用同一个工具导致的死循环和 Token 浪费问题。

- **v1.0.2** **绘图能力初探**
  - 买了一只笔，学会了画图，底层新增图片自动捕获机制，优化了代码结构。

- **v1.0.1** **基础体验修复**
  - 修复了沙箱中没有 AstrBot 的 bug (加了个图片 Logo)。

- **v1.0.0** **初始版本**
  - 在与 AI 评审员的深度交流中茅塞顿开，推出了一个能跑的版本，基础代码执行、隔离运行环境正式上线。
